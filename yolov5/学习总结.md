# YOLOv5 结构三部分学习总结（详细版）

## 1. Backbone：CSPDarknet

### 1.1 这部分到底在做什么

我现在把 backbone 的任务定义为：

- 把输入图像从“像素空间”变成“语义特征空间”
- 逐层压缩空间分辨率、提升通道表达能力
- 输出多尺度特征给 neck 做融合

它不直接负责给出最终检测框，而是提供检测所需的基础表征。

### 1.2 为什么是 CSPDarknet，而不是直接堆 ResNet

核心思想是 CSP（Cross Stage Partial）：

- 把输入特征分两路
- 一路做较深变换（多个 Bottleneck）
- 一路保留较浅信息（较少变换）
- 最后再 `concat + conv` 融合

我理解这样做的收益有三点：

- 降低重复梯度和冗余计算
- 保留原始特征流，减轻深层变换带来的信息覆盖
- 在计算量可控前提下提升特征表达

### 1.3 关键模块逐个看

`CBS`（Conv + BN + SiLU）

- Conv：线性可学习变换，抽取局部模式
- BN：稳定分布、加速收敛
- SiLU：平滑非线性，梯度性质通常比 ReLU 更柔和

`C3`（YOLOv5 中 CSP 的核心载体）

- 外层是分流与聚合
- 内层通过 Bottleneck 叠加提取深层语义
- 与 ResNet 同样强调“信息捷径”，但融合方式是 `concat` 而不是 `add`

`Bottleneck`

- 小残差单元
- 负责在深分支内持续提炼高层特征
- 可以理解为“局部特征加工器”

`SPPF`

- 位于 backbone 末端
- 用池化方式做多尺度上下文聚合
- 主要目的是扩大有效感受野，强化进入 neck 前的高层语义

### 1.4 add 和 concat 的本质区别

`add`（ResNet 风格）

- 逐元素相加
- 需要形状一致
- 更偏“残差修正”

`concat`（CSP/C3 风格）

- 按通道拼接
- 保留两路完整信息后再交给卷积学习融合
- 更偏“信息保留 + 后续学习融合”

### 1.5 Backbone 的 shape 变化规律

典型检测 backbone 一般遵循：

- 空间尺寸：`H x W` 逐步减小
- 通道数：`C` 逐步增大
- 表征属性：低层偏细节，高层偏语义

所以我会把它记成：

输入图像
-> CBS（初步提特征 + 降采样）
-> 多层 C3（语义逐层增强）
-> SPPF（上下文增强）
-> 输出多尺度特征图

## 2. Neck：带 C3 的 PAN/FPN 融合层

### 2.1 Neck 的目标

我理解 neck 的核心目标是“跨尺度融合”：

- 让高层语义流向低层（补语义）
- 让低层定位流向高层（补细节）
- 最终得到语义和定位更平衡的多尺度特征

### 2.2 FPN + PAN 的分工

FPN（自顶向下）

- 上采样高层特征
- 与浅层特征拼接
- 提升小目标所需的语义信息

PAN（自底向上）

- 下采样低层融合结果
- 再与高层特征拼接
- 强化定位路径和结构信息传递

### 2.3 为什么 neck 里还要 C3

每次拼接后直接输出通常不够稳定，YOLOv5 会在融合后接 C3/CBS 进一步整合，作用是：

- 降低拼接后通道冗余
- 学习更有效的跨层组合关系
- 让最终给 head 的每个尺度特征都更“可检测”

### 2.4 Neck 的一句话流程

backbone 多尺度输出
-> 顶到底（FPN）融合
-> 底到顶（PAN）再融合
-> 融合点用 C3 提炼
-> 送入三个检测尺度

## 3. Head：Detect Head（详细）

### 3.1 位置和任务

head 位于 backbone + neck 之后，输入是 neck 输出的三个尺度特征图。  
例如常见检测尺寸可对应：

- `80 x 80`：偏小目标
- `40 x 40`：偏中目标
- `20 x 20`：偏大目标

head 的任务是把“特征表示”转成“检测结果表示”：

- 框的位置和大小
- 目标存在置信度
- 类别概率

### 3.2 Head 输入张量怎么理解

每个尺度输入形状写成：`B x C x H x W`

- `B`：batch size
- `C`：通道数
- `H, W`：网格大小

我现在的理解是：每个 cell 都可看作一个局部检测单元，负责它感受野附近的目标预测。

### 3.3 1x1 Conv 做参数映射

head 在每个尺度上先做 `1x1 Conv`，这一步重点是“映射到预测参数空间”，不是继续做高层特征抽象。

输出形状：

`B x (na * (5 + nc)) x H x W`

- `na`：每个 cell 的 anchor 数，YOLOv5 常用 3
- `nc`：类别数
- `5`：`x, y, w, h, obj`

若 `nc=80`：

- 每个 anchor 输出 `85`
- 每个 cell 总输出 `3 * 85 = 255`
- 即 `B x 255 x H x W`

### 3.4 reshape 后语义才完全清楚

将通道展开成 anchor 维度：

`B x 3 x H x W x (5 + nc)`

此时每一项 `(b, a, i, j, :)` 就是：

- 第 `b` 张图
- 第 `a` 个 anchor
- 第 `(i, j)` 个网格位置
- 对应的一整组检测向量

### 3.5 单个预测向量包含什么

向量形式：

`[x, y, w, h, obj, cls1, ..., clsnc]`

含义：

- `xywh`：框参数（网络输出空间中的参数，不是最终像素坐标）
- `obj`：目标存在性置信度
- `cls`：类别分数分布

### 3.6 为什么必须 decode

网络直接输出的 `xywh` 不是原图绝对框，需要借助先验信息还原。

decode 时会结合：

- grid（当前网格坐标）
- stride（当前尺度相对原图步长）
- anchor（当前尺度先验宽高）

YOLOv5 常见思路可概括为：

- `x, y` 学习相对网格偏移，再映射到图像坐标
- `w, h` 学习相对 anchor 的缩放，再映射到真实宽高

最终得到候选框中心宽高 `(bx, by, bw, bh)`，再可转成角点框 `(x1, y1, x2, y2)`。

### 3.7 三个尺度做同一套流程

三个尺度并行执行：

特征图
-> `1x1 Conv`
-> reshape
-> 拆分 `xywh/obj/cls`
-> decode

三者差异在：

- `H, W` 不同
- stride 不同
- anchor 组不同

因此形成对小/中/大目标的互补检测能力。

### 3.8 后处理为什么必要

decode 后候选框数量通常很大，且包含大量低质量、重复框，需要后处理：

- 低分过滤（按 `obj` 或融合分数）
- 类别置信度融合（常见可理解为 `score = obj * cls`）
- NMS 去重（保留高分，抑制高 IoU 重叠框）

推理端最终链路：

候选框
-> 置信度筛选
-> NMS
-> 最终检测结果

### 3.9 训练和推理要严格区分

推理阶段：

- 关注最终输出可读框
- 会执行 decode、筛选、NMS

训练阶段：

- 重点是优化参数，不需要 NMS 参与反传
- 使用 head 原始预测张量与 GT 做匹配并计算损失

这一点很关键：NMS 是后处理，不参与梯度反传。

### 3.10 训练时 head 主要学什么

loss 通常拆成三部分：

- box loss：框回归质量（常结合 IoU 系列度量）
- obj loss：目标存在性判断是否正确
- cls loss：类别判断是否正确

总损失是加权和，通过：

前向预测
-> 损失计算
-> 反向传播
-> 优化器更新

不断让 head 学会把多尺度特征映射成更准确的检测结果。

### 3.11 Head 的浓缩结论

YOLOv5 head 的本质是：在三个尺度上并行完成“参数预测 -> 框解码 -> 后处理输出”；在训练中通过 `box + obj + cls` 联合优化，使模型逐步学会更准确地定位、判定目标存在性并完成分类。

## 4. 一页式总链路

输入图像
-> Backbone（CSPDarknet）提取多尺度语义特征
-> Neck（PAN/FPN + C3）做跨尺度双向融合
-> Head（Detect）输出 `xywh + obj + cls`
-> Decode + 筛选 + NMS
-> 最终检测框
